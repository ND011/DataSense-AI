DataSense-AI System Analysis

Overview
DataSense-AI is an end-to-end web application intended for autonomous dataset analysis, type detection, and interactive visualization. The platform enables users to upload CSV or Excel files and instantaneously receive a full statistical profile, visualization recommendations, business inferences, and predictive insights, without requiring them to manually configure their views.

Architectural Breakdown
The application is structured into two main components:
1. Frontend (/frontend): A React 19 application built via Vite, leveraging Tailwind CSS for its styling. It handles user file uploads and renders interactive dashboards utilizing libraries like Recharts for graphs and visual layouts.
2. Backend (/backend): A robust FastAPI service powered by Pandas, Numpy, and an local LLM (via Ollama SDK). It orchestrates a multi-step analytical pipeline from data ingestion to insight synthesis.
3. Standalone Scripts (/): Root scripts like annotation.py and charts.py primarily serve as offline, batch testing variants of the pipeline engine logic.

The Analytical Pipeline (How it works internally)
When a user uploads a dataset via the frontend, the FastAPI backend endpoint /analyze (backend/main.py) processes the file through a defined 17-phase pipeline:

1. Data Ingestion & Deduplication
The uploaded CSV is loaded into memory using Pandas (pd.read_csv). The system ensures clean headers by deduplicating any identical column names sequentially (e.g., value, value_1).

2. Initial Column Type Detection (column_detector.py)
This is the heart of the engine's autonomy. It classifies each column into strict typologies without human input:
- Identifiers: Columns tagged via keywords (id, uuid, code) or exceptionally high uniqueness (>95%). These are aggressively ignored in downstream maths and charts.
- Geographic: Columns detected by keywords like country, latitude, postal_code, triggering map visuals.
- Datetime: Deduced by column names (year, date) or successful pandas parsing.
- Numeric (Continuous/Discrete): Based on numeric data types, further split by unique value counts (>20 vs <=20).
- Categorical & Binary: Text columns with a reasonably low set of discrete values (<=15).

3. Early Gate: Lookup Table Check
If the dataset appears to just be a reference mapping (e.g., 2 columns representing ID -> Country Name), the engine prudently halts visualization and complex ML, returning a simple tabular view.

4. Data Cleaning & Normalization (data_cleaner.py / categorical_normalizer.py)
- Standardizes capitalization.
- Flags and handles outliers using IQR metrics.
- Normalizes minor categorical variations and missing data to prepare a solid baseline dataset.

5. Feature Engineering
The engine simulates new attributes where logically possible, such as computing an individual's Age from a detected DOB or Birth_Year column.

6. Statistical Profiling (dataset_profiler.py)
It computes a deep mathematical profile across the clean dataset. This captures distributions, median, standard deviations, skewness, missing value ratios, and categorical frequency clusters.

7. Domain Detection (business_advisor.py)
Using both heuristic keywords and AI semantic analysis (via Ollama), the dataset is typed into business domains like Finance, Healthcare, E-Commerce, or HR. 
A specific domain "Analyzer" class is booted up to ensure that insights and KPIs are bespoke to the industry context.

8. Visualization Recommendation (chart_recommender.py)
A rules-engine assigns the most informative chart type based on column relationship pairs:
- Continious vs Continuous -> Scatter Plot
- Categorical vs Continuous -> Box Plot
- Single Continuous -> Histogram
- Geographic -> Point map / Choropleth
The engine prioritizes maps and domain-specific golden paths first, falling back to statistical rules for standard combinations.

9. Chart Data Aggregation (chart_data_generator.py)
The raw dataframe is grouped, binned, and minimized down to just the nodes needed for the frontend React components to render the recommended visuals cleanly.

10. Pivot & Insight Generation (insight_generator.py)
Scans the statistical summaries to build automated bullet points. Examples include "Most common category in Region is North America (43%)" or pointing out extreme numeric outliers.

11. Predictive Modeling (predictive_modeler.py)
A fast ML simulation is performed on viable target columns (like continuous numeric columns representing cost/sales), fitting basic linear regression or classification models to provide forward-looking inferences.

12. Generative AI Synthesis (llm_service.py)
Finally, the analytical metadata (profiling stats, top insights, domain, and KPI suggestions) are pumped into a local LLM via Ollama to write a cohesive, human-readable "Executive Summary" for the dashboard.

Overview Point Of View

From an engineering perspective, DataSense-AI is a highly modular, rule-based expert system wrapper combined with local generative AI.

It heavily leans on Heuristics (hard-coded analytical rules of thumb) rather than complex deep learning for its structural decisions. This makes it incredibly fast, deterministic, and easily adjustable. 

Because it intelligently tags identifiers and actively routes metrics to domain analyzers, it successfully bridges the gap between generic "Auto-EDA (Exploratory Data Analysis)" scripts and an actual business-intelligence product. The architecture enforces separation of concerns—data crunching is isolated in the FastAPI backend, while the Vite frontend focuses exclusively on interactive state and rendering the payload.
